# Verto

Advertorial magazine site built on Astro 5, deployed to Cloudflare Pages. Promotes affiliate offers via problem-solving landing pages — optimized for Facebook page/group traffic.

## Stack

- **Astro 5** — static site generator
- **Tailwind v4** — CSS via `@tailwindcss/vite`
- **Cloudflare Pages** — hosting
- **MaxBounty** — affiliate network

## Structure

```
src/
  data/
    offers.ts          # All 34 affiliate offers with metadata
    problems.ts        # 20 problem definitions (solve pages)
    categories.ts      # Site categories
    config.ts          # Affiliate config + URL helpers
    tracking-links.json  # Auto-generated by fetch-links script
  pages/
    index.astro        # Homepage — problem grid + top offers
    solve/[slug].astro # Problem landing pages (Facebook ad targets)
    offer/[slug].astro # Individual offer review pages
    offers/index.astro # Browse all offers
    category/[slug]    # Category hubs
  components/
    OfferCard.astro    # 3 variants: featured, full, compact
```

## Scripts

```bash
npm run dev          # Local dev server
npm run build        # Build to ./dist
npm run fetch-links  # Pull tracking links from MaxBounty API
npm run discover     # Scan MB for new campaigns not yet in catalog
npm run deploy       # fetch-links → build → deploy to Cloudflare Pages
```

## Setup

1. Copy `.env.example` to `.env` and add your MaxBounty credentials:
   ```
   MB_EMAIL=you@maxbounty.com
   MB_PASSWORD=yourpassword
   ```

2. Run `npm run fetch-links` to populate `tracking-links.json` with real tracking URLs.

3. Run `npm run deploy` to build and push to Cloudflare.

## Adding New Offers

1. Add the offer object to `src/data/offers.ts`
2. Add its `id` to the `OFFER_IDS` array in `scripts/fetch-tracking-links.mjs`
3. If it covers a new problem, add a `Problem` entry to `src/data/problems.ts`
4. Run `npm run fetch-links && npm run build`

## Affiliate Tracking

Tracking links are fetched via the MaxBounty API before each build and stored in `tracking-links.json`. If a link can't be fetched, the offer falls back to its `landingPage` (no commission). Priority order:

1. Auto-fetched API link (has proper `k=` key, full tracking)
2. Manually set `trackingLink` on the offer
3. `landingPage` — no commission, fallback only

Sub-ID attribution: every link gets `&sub1=[problem-slug]` appended so MaxBounty reports show which solve pages are driving conversions.
